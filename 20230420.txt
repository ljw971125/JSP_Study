내장객체

request 객체

    클라이언트와 서버에 대한 정보 읽기
    클라이언트가 전송한 요청 매개변수에 대한 정보 읽기
    요청 헤더 및 쿠키 정보 읽기

    user-agent : 웹 브라우저의 종류를 알 수 있습니다.
    referer : 리퍼러는 웹을 서핑하면서 링크를 통해 다른 사이트로
    방문시 남는 흔적을 말합니다.

    cookie : 요청 헤더를 통해 쿠키도 확인할 수 있습니다.

response 객체
    
    add계열 메소드 : 새로운 헤더명으로 값을 추가합니다.
    동일한 헤더명이 있으면 동일한 이름으로 값을 추가합니다.

    set계열 메소드 : 기본의 헤더값을 수정합니다.
    단. 동일한 헤더명이 존재하지 않는다면 새롭게 추가합니다.

out 객체

out 내장 객체는 웹 브라우저에 변수 등의 값을 출력할 때 
주로 사용합니다.

application 객체
    
    application 내장 객체는 웹 애플리케이션당 하나만 생성되
    며 JSP 페이지에서 접근합니다.

exception 객체
    
    예외 처리를 위해 사용합니다.

    HTTPS       에러 의미                               조작방법
    에러코드
    404         Not found: 클라이언트가 요청한          URL 상의 경로명이나 파일명이 제대로 입력되었
                경로에서 문서를 찾을 수 없습니다.        는지 확인합니다.

    405         Mathod Not Allowed: 허용되지 않는      주로 서블릿으로 개발할 때 발생하게 되는데
                메서드라는 뜻으로 GET방식 혹은 POST     doGet(), doPost()메서드가 적적히 오버라이딩
                방식으로 요청했는데 이를 처리할          되었는지 확인합니다.
                컨트롤러가 없습니다.

    500         Internal Server Error : 서버 내부       가장 많이 발생하는 에러로, 개발 중인 코드를

                문제가 있어 웹서버가 요청 사항을 수행
                할 수 없습니다.

내장객체의 영역(scope)

page 영역 : 동일한 페이지에서만 공유됩니다.
            페이지를 벗어나면 소멸됩니다.

request 영역 : 하나의 요청에 의해 호출된 페이지와
              포워드(요청 전달)된 페이지까지 공유됩니다.

session 영역 : 클라이언트가 처음 접속한 후 
               웹 브라우저를 닫을 때 까지 공유됩니다.

application 영역 : 한 번 저장되면 웹 애플리케이션이
                  종료 될 때까지 유지됩니다.
